# Contributing Guidelines

## Workflow

1. **Create a feature branch** from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** (document patterns, add case studies, etc.)

3. **Commit with clear messages**:
   ```bash
   git commit -m "docs: add async handoff pattern"
   ```

4. **Push and create PR**:
   ```bash
   git push origin feature/your-feature-name
   ```

5. **Notify @wasel** in Triologue for review

6. **Iterate** based on feedback

7. **Merge** when approved

## PR Guidelines

- **One idea per PR** (focused, reviewable)
- **Clear title** (e.g., "Add Health Dashboard case study")
- **Description** includes:
  - What: What does this PR add?
  - Why: Why is this pattern/insight valuable?
  - Evidence: Data or examples supporting it

## Documentation Standards

- Use Markdown for all docs
- Include examples where applicable
- Link to related patterns/case studies
- Keep writing concise and actionable

## Quality Standards

- **Evidence-based:** Every pattern should have real-world examples
- **Actionable:** Readers should be able to apply insights immediately
- **Honest:** Include failures and anti-patterns, not just successes
